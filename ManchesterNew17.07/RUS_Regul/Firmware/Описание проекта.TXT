1. Проект: РУС регулятор на базе контроллера STM32L433CCT3 (LQFP48)
2. Общий принцип дейтсвия:
	- в модуле пристутсвуют три платы регулятора, отвечающие за три выдвижные "лапы"
	- каждая плата имеет свой адрес, который определяется исходя из состояние двух адресных пинов
	- у каждой платы две основные функции: измерение давления и управление мотором
	- к редуктору мотора прикреплен "ротор", который вращяясь относительно "статора" регулирует давление на поршень. При достижении определенного давления, "лапа" 
	  выдвигается, упираясь в стенку скважины, толкая всю телесисему в противоположную вектору давления сторону ("лапа" - выдвижная пластина, упирающаяся в стенку скважины).
	- управление регуляторами осуществялет плата телеметрии, которая расчитывает результирующий вектор и выставляет необходимое выдвижение (усилие) каждой из трех "лап", которые
	  направляют телесистему в нужную сторону
	- на 08.04.2024 модуль получает питание с генератора (+12 В) только когда есть поток. Если потока нет, генератор не работает, питание на модуле нет
	- для того, чтобы задвинуть "лапы" при внезапном (и не только) отключении питания, в конструкции каждой лапы пристутсвует возвратная пружина.
	- модуль РУС не связан проводами ни с одним модулем телесистемы. Связь с телеметрией РУСа по электромагнитному каналу (через породу) 
	- управление мотором
		- установка скорости
		- установка заданного угла поворота
		- ориентация в пространсвте (калибровка)
		- установка процента перекрытия
		- управление направлением вращение
		- измерение скорости вращения
		- управление питанием мотора
	- измерение давления
	- измерение тока потребления
	- измерение входного питания
	- обмен с платой "Контроллер телеметрии" по протоколу SKLP
	- определение собсвтенного адреса (для каждой платы свой)
	- измерение температуры контроллера
	- вспомогательный UART для отладки (к функционалу почти не относится, висит на контрольных точках)
	- обработка данных АЦП через DMA
	- обработка посылок SKLP через DMA (по сути входит в библиотеку SKLP)
	- операционная система реального времени (freeRTOS)
4. Используемая периферия контроллера:
	- "Хардовые" таймеры:
		- TIM1 - таймер в режиме ШИМ. Управление скоростью мотора путем коэффициента заполнения ШИМ-сигнала. Для мотора CM36_3650: чем больше заполнение - тем быстрее скорость.
		- TIM2 - таймер в режиме ETR2 (external trigger). Счет скорости вращения мотора. Считает импульсы обратной связи с мотора. 
		- TIM6 и TIM7 таймеры контроля таймаута посылок по SKLP для UART'ов (*УТОЧНИТЬ!)
	- Порты вводы-ввыода:
		- порт на транзисторный ключ для управления питанием
		- порт для изменения направления двжиения мотора (высокий либо низкий уровень соотвествует вращению по/против часовой стрелки)
		- два адресных порта
		- вкл/выкл внешнего АЦП
		- порт выбора скорости внешнего АЦП
	- Каналы АЦП
		- внутренний датчик температура
		- внутренне опроное напряжение (Vref)
		- канал для оцифровки встроенного операционного усилителя (OPAMP)
		- канал для оцифровки напряжения питания через делитель
	- Внешний АЦП для измерения давления ADS1231. Обмен c ADS1231 по протоколу SPI
	- Операционный усилитель (OPAMP) для измерения тока потребления. Сигнал с шунта (0.1 Ом) передается в на операционник (Vinp), где пересчитывается в ток, а потом оцифровывается внутренним АЦП
5. Используемые микросхемы:
	- LTC3630MSE (название уточнить) для получения +3.3 В цифрового и аналогового питания из + 12 В
	- TPS.... Для получения +5 В цифрового и аналогового питания из + 12 В
	- Буферы для преобразования уровня (сигнал обратной связи, направление двжиения и ШИМ (скорость): 0-5В)
